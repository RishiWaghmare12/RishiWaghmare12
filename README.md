## About Me

I'm a computer science student at MIT Academy of Engineering, Pune, who likes building things end to end — database schema, API, UI, and deployment. My repos lean toward practical, full-stack applications: warehouse and inventory systems, a hospital management platform, and machine learning projects that end up wrapped in a working web app rather than just a notebook.

A few things that show up across my work:

- I default to a **React + Node/Express + PostgreSQL or MySQL** stack for web apps, and I care about clean project structure (separate `Backend`/`FrontEnd` folders, `.env.example` files, health-check endpoints).
- On the ML side, I don't stop at training a model — I compare multiple algorithms (Naive Bayes, KNN, SVM, ensembles) and ship the winner behind a **Streamlit** interface.
- I'm getting comfortable with the DevOps side too: containerizing apps with **Docker**, wiring up **Jenkins** pipelines, and deploying to **Vercel**.
- I write documentation the way I'd want to read it — architecture diagrams, request flows, and setup steps included, not an afterthought.

### 🏭 [DBMS-Warehouse-System](https://github.com/RishiWaghmare12/DBMS-Warehouse-System)

A production-style, full-stack warehouse management system for tracking inventory across categorized compartments with live capacity checks and a complete transaction audit trail.

**Stack:** React 19, Vite, Express, PostgreSQL, Axios, Vercel

**Highlights**
- Real-time capacity tracking with utilization indicators per compartment
- Transactional send/receive workflow with capacity enforcement at both compartment and item level
- Auto-seeding demo data plus a `/health` endpoint for deployment monitoring
- Deployed live on Vercel with a documented REST API

**Why it stands out:** it's not a toy CRUD app — it models real warehouse constraints (capacity limits, audit logging, rollback on failed transactions) with a genuinely clean separation between backend and frontend.

### 📊 [AIML-project-MDM](https://github.com/RishiWaghmare12/AIML-project-MDM) — Customer Churn Prediction

An end-to-end ML project predicting telecom customer churn, from raw data to a deployed prediction app.

**Stack:** Python, scikit-learn, pandas, Streamlit, JupyterLab, uv

**Highlights**
- Compared 8 models — Naive Bayes, KNN, SVM, Decision Tree, Logistic Regression, plus Voting, AdaBoost, and Stacking ensembles
- Selected the Voting Ensemble (F1-score 0.6164) as the best trade-off given class imbalance
- Packaged the winning model and scaler for a live [Streamlit app](https://aiml-project-mdm.streamlit.app)

**Why it stands out:** the model selection is justified with real metrics and reasoning (F1 over raw accuracy, given class imbalance) rather than just picking the highest-accuracy option.
